The Pipeline — Step by Step
1

Document Parsing

PDFs, DOCX, images, and 14 other formats are parsed into clean text. Forms are extracted with filled field values. Images use built-in OCR.

2

Smart Chunking

Text is split into overlapping chunks (~1000 chars) with contextual prefixes: filename, page, and section headers prepended for retrieval accuracy.

3

Embedding

Each chunk is converted to a 384-dimensional vector using BGE-small-en-v1.5 (local ONNX model). Asymmetric encoding: queries and documents are embedded differently for better matching.

4

Hybrid Retrieval

Your question is searched two ways simultaneously — BM25 (keyword matching with legal synonyms) and cosine similarity (semantic meaning). Results are fused via Reciprocal Rank Fusion, then deduplicated and reranked for diversity using MMR.

5

LLM Generation

Top-ranked chunks are assembled into a prompt for Qwen3-8B (running locally via llama.cpp). The model generates a grounded answer with inline citations. Post-processing strips hallucinations and validates every citation.

Hybrid Retrieval — Two Scorers, One Ranking

BM25 (Keywords)

"What is the termination clause?"
Finds exact term matches with legal synonym expansion

+

Cosine (Semantic)

"How can either party end the agreement?"
Finds meaning-similar passages via vector distance

Reciprocal Rank Fusion (RRF)

Merges both rankings into a single, calibrated result list — robust across query types
